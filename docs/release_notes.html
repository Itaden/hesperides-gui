back 1.0.0 - front 1.0.2 [NEW]

[HES-235] global properties are not displayed well (after scrolling) Bug Fix

If you have a lot of global properties, when you scroll the display has not a good behavior.

[HES-236] Problem for message displayed when module does not exist Bug Fix

Sometimes the message "At least one empty module has been detected on the current platform." does not appear well.

[HES-238] Users settings : you should not be able to add empty applications Bug Fix

It should be forbidden to add an empty appliaction.

[HES-246] Toggle switch "Show unfiled properties" stay enabled Bug Fix

When you toggle "Show unfiled properties" on a module with unfilled properties and then switch to another whose does not have unfilled properties, the switch stays enabled and locked. You have to swith to the previous one and disable the switch to see module props.

[HES-247] link top left does not work on firefox Bug Fix

If you click on "Hesperides" top left on the screen, you should go on welcome page : OK for Chrome, KO for firefox

back 1.0.0 [NEW] - front 1.0.0 [NEW]

[HES-115] Stuck in loop in diff mode when the platform doesn't exist Bug Fix

The start diff buton will remain inactive while the application name doesn't have any platform, but also while the choosen plateform doesn't belong to clickable list.
In order to improve the browsing experience, a previous button has been added during the diff between module to avoid "Cancel and start again" actions.

[HES-130] Writing a comment when editing the valuation of properties New Feature

From now on, when the valuation of properties changes, writing a comment will be mandatory, in the same way we fill out comments on versionning tools. The added value of this feature is the ability of retrieving a comment from the platform history, and automatically ease the task of finding out what have been edited. In case of that there is no changes but still trying to save properties, a warning pop-up inform that no changes have been detected.

[HES-133] Obtain information about number of applications/platforms/modules New Feature

The pupose is to create a REST service to obtain different kind of stats around hesperides.

[HES-144] Saving users preferences New Feature

A user preferences system has been put in motion to memorize some parameter in order to customize web interface for each user. These parameters are stored locally. Its allows to :
  • Memorize the default language
  • Setup default type of view between tree and bloc mode
  • Configure the color of platforms
  • Define favorites apps to access them in a single clic from the home page Hesperides

[HES-149] Hide password properties in production platform New Feature

For a production platform, if you are a non authorized user, properties with "@password" annotation should be displayed with "********". (be careful to consider front and back access).

[HES-153] Find applications using a particular module New Feature

From the module editing screen, it is now possible to visualize which applications are using this module, making their access easier by clicking a link. It also allows to see which applications will be impacted by changes on the module.

[HES-154] Show / Hide the platform ribbon on the application view New Feature

The platform ribbon can be hidden or displayed in order to save space on screen for small devices or simply for applications holding a lots of platforms.

[HES-165] Activating Nexus Mode Bug Fix

The Nexus mode allows, at the application level, to apply a delivery note configuration. This action will find the module present at the same time on the application and the delivery note, and applies to each of them the corresponding module version number specified in the delivery note
A SNAPSHOT delivery note will force the usage of a working copy of the module, while in the opposite case, it will apply a released one.
The nexus mode can apply module version number that aren't already created or even deleted, in this case a red indicator will help you find them. The valuation of the deleted modules appear as "deleted", but to retrieved them all, you only have to create the targeted module.

[HES-171] Add button "Clean useless valuation" on the in the instance properties New Feature

A button has been added to delete useless valuation. This button was only at the module level, from now on we can use it at the instance level.

[HES-175] Filter displaying only deleted properties Bug Fix

The deleted properties filter was already displaying deletes ones, but instead of displaying only the deleted properties, they were displayed among the other as crossed. Now, this filter behave as its peer of non filled properties, meaning, displaying only the corresponding properties.

[HES-176] Find global properties usage New Feature

When we display global properties, it is now possible to see where these properties are used in the valuation. A small eyes icon appears next to the remove icon if this properties is used. Obviously, when this icon isn't displayed it mean that is properties isn't not used among the application. Clicking the eyes icon drop down the list of usage location.

[HES-178] Preview of generated files path Bug Fix

Until now in the preview of generated files a small bug was preventing the path from displaying. This bug has been fix, the path is now displayed correctly.

[HES-179] Event filtering in platform history New Feature

Two new filters has been added :
  • Search input : this field allow to filter on authors, modules, versions, dates and titles
  • "My History" checkbox : when activated, are visible only the events of the connected user
For performances reasons, filters are cleared when the user request more event through the button "More".

[HES-180] Wording change on the platform save event in history Improvement

Before, in the history, no module name was displayed, only "XXX modified platform properties". From now on, the module name is also displayed to ease the readability.

[HES-185] Confirmation popup added before running "Clean useless valuation" Improvement

A confirmation pop has been added before cleanning useless valuation to prevent unwanted actions. Be carefull, it is mandatory to save properties in order to apply definetly these modifications.

[HES-186] Fix applications auto-complete Bug Fix

After a research from the auto-complete, we were left with residual string name of the application, therefor

[HES-187] Do not copy events when you copy a platform from another Improvement

When created a platform from another, the new platform contains all the events of the old. And events are recorded at the date and time of creation of the platform.
It is preferable to have a new event to get the old events.

[HES-188] Open diff in a new tab Improvement

Open diff a new new tab to easily go back to the valuation screen.

[HES-189] Compare two events New Feature

Possibility to compare two events to know exactly what was the modification of valuation.

[HES-190] bug when we want to change platform version with nexus mode Bug Fix

When you want to change the value of a platform version, Hesperides proposes to extract and apply the Hesperides versions of modules corresponding NDL.
When the NDL contains packages with defined HesperidesVersion and different from the version of the package, Hesperides applies package version and not the HesperidesVersion.
Hesperides must apply version hesperides module in this case.

[HES-193] When we run a diff, properties containing a templated value are not displayed Bug Fix

Templated values are not considered well when we compare two platforms.

[HES-194] go back to the right screen after a diff New Feature

Duplicates [HES-144].

[HES-195] impossible to use diff with date picker Bug Fix

Date picker does not considered well the today's date to allow a diff at an earlier date.

[HES-196] quick access to display entirely the platform tree New Feature

With tree mode, it would be nice in one click to open trre entirely.

[HES-203] regression for autocomplete engine Bug Fix

Regression seen on new version : autocomplete with a dash (particularly) does not return right results.

[HES-205] add id for web element Improvement

To create protractor tests, we have to add id on web elements.

[HES-207] Sort button for global properties is visible even if global properties are hidden Improvement

The sort button for global properties have to be visibleonly if global properties are displayed.

[HES-210] [PREVIEW] impossible to preview files if a global property contains "$" Bug Fix

If a global property contains "$FOO", it's then impossible to preview files. An error message appears : "illegal group reference".

[HES-211] Impossible to use a global property on instance level Bug Fix

A global property is not recognize on instance level : so it's not possible to use this feature.

[HES-212] [SWAGGER] some services are not well documented Bug Fix

bug about documentation of swagger.

[HES-213] Add protractor tests Improvement

For main features, add some protractor tests.

[HES-221] Switch "show unfilled properties" does not work like other switch Bug Fix

When switch "show unfilled properties" is on and you start to fulfill a property, this property disappears because the system assumes that the property is not fulfilled.

[HES-223] "save global property" button does not work like "save" in module or instance level Bug Fix

Currently, to save global property you press "enter" and to save a removal you click on "save" button. Those behavior should be aligned with module/instance level.

[HES-224] Display which pattern is used when you valuate properties New Feature

Currently, to save global property you press "enter" and to save a removal you click on "save" button. Those behavior should be aligned with module/instance level.

[HES-229] [PRODUCTION MODE] prohibit an unauthorized user to create a platform from a production platform Improvement

a non authorized user should not be able to create a platform from a production platform to prevent the recovery passwords.
